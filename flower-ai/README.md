# Flower AI — Flower Classification + RAG Q&A

Upload a flower photo, get a prediction with a confidence score, see a quick
reference card about the flower, and ask follow-up questions answered by a
RAG pipeline (Chroma + Gemini) over your own PDFs.

## Project layout

```
app.py                  Streamlit UI — coordinates everything below, has no ML/RAG logic itself
ingest.py                One-time script: builds vector_db/ from your PDFs
src/predictor.py         Loads the .h5 model, preprocesses images, runs inference
src/rag_pipeline.py      Retrieves from Chroma, asks Gemini, returns answer + sources
src/utils.py              Small shared helpers
data/class_names.json    Must match your model's output order exactly
data/flower_info.json    Static info shown in the "About <Flower>" card
data/knowledge_base/      Drop your PDFs here before running ingest.py
model/flower_classifier.h5   Your trained model (not included — see below)
vector_db/                Generated by ingest.py — commit this to GitHub, don't regenerate on Cloud
```

## First-time setup

1. **Create a virtual environment and install dependencies**
   ```
   pip install -r requirements.txt
   ```

2. **Add your trained model**
   Copy your `flower_classifier.h5` from Colab into the `model/` folder.

3. **Check `data/class_names.json`**
   The order of names in this file must exactly match the order your model
   was trained with (usually alphabetical, if you used Keras's
   `image_dataset_from_directory` or `ImageDataGenerator`). The placeholder
   list here has 10 example flowers — replace it with your real classes if
   they differ, in the same order.

4. **Get a Gemini API key**
   Free at [Google AI Studio](https://aistudio.google.com/app/apikey). Copy
   `.env.example` to `.env` and paste your key in.

5. **Build the RAG knowledge base**
   Drop your flower PDFs into `data/knowledge_base/`, then run:
   ```
   python ingest.py
   ```
   This only needs to be re-run when you add or change PDFs — not every time
   you start the app.

6. **Run the app**
   ```
   streamlit run app.py
   ```

## Deploying to Streamlit Cloud

- Commit everything, **including `vector_db/`**. Streamlit Cloud's filesystem
  doesn't persist reliably between restarts, so the app should never need to
  re-run `ingest.py` (and re-spend API quota) on a cold start — it should ship
  with the database already built.
- In the app's **Settings → Secrets**, add:
  ```
  GOOGLE_API_KEY = "your_real_key_here"
  ```
- Point the deployment at `app.py`.
